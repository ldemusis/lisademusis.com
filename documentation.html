Documentation

Work Type:Process, Requirements
Company:Various

Documentation is beneficial for a number reasons: It creates an accessible source for teams to reference product objectives, it can outline scope and specifications before being committed to tickets, and it has the ability to improve understanding and accountability across teams and departments. Below are two project examples where I created documentation as foundational support to product development and project management.

RevZilla Ecommerce

At RevZilla, during a complete replatform and redesign, documentation was leveraged to establish a lexicon for the project. With new features popping up frequently, many departments needed a guide to reference when placing design, marketing, or development requests.
